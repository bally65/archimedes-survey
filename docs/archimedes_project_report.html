專案概覽

Archimedes 是一套完全自製的自主調查機器人系統，專為在彰化西海岸泥灘地執行美食奧螻蛄蝦棲地密度調查而設計。

🎯

研究目標

自動化偵測並記錄美食奧螻蛄蝦（Austinogebia edulis）洞穴位置與密度，建立彰化海岸棲地健康指標，取代耗時耗力的人工樣方調查。

📍

作業地點

彰化西海岸 5 個調查區：王功（41格）、漢寶（35格）、伸港1（29格）、伸港兩保育區之間（20格）、伸港2（26格）。總計 151 格，102.5 公頃。

🤖

技術創新

全球首例以阿基米德螺旋推進機器人執行潮間帶底棲生物聲學普查。結合 200kHz C-scan 超音波、YOLOv8 視覺、GPS/IMU 自主導航與 Deep RL 控制。

💰

成本優勢

單機採購成本約 NT$6,741（路線A步進馬達版）。5 台艦隊總採購成本估計 ~NT$33,705，遠低於商業調查設備。

📅

開發時程

2026年3月開始硬體採購，目標 2026年5月進行第一次王功野外部署。最佳調查窗口：3–5月、9–11月（水溫 22–29°C）。

📊

科學貢獻

建立機器人密度圖 vs 人工樣方驗收框架（RMSE、r²、Moran's I、Bland-Altman），目標 ROC-AUC ≥ 0.80，達到可投稿學術期刊水準。

目標生物：美食奧螻蛄蝦

Austinogebia edulis — 俗稱鹿港蝦猴，是彰化泥灘地的關鍵生態工程師。

🚨
計數關鍵：Y 型洞穴有 2 個開口！
每隻奧螻蛄蝦建造 Y 型洞穴，有 2 個地面開口，相距 21–26cm。自動偵測時必須將 30cm 內的開口配對，再除以 2 才得到個體數量。本系統已實作配對邏輯（is_new_individual 欄位）。

⚠️

DMOMP 頻散補償

# 線性頻散模型：c(f) = c₀ + β × (f - f₀) / f₀
# 彰化粉砂：c₀ ≈ 1500 m/s，β = 15–20，f₀ = 200kHz
# 相位校正：Δφ = ω × 2d × (1/c₀ - 1/c(f))
# 效果：深度目標 TOF 誤差從 ±5ms 降至 ±0.3ms

導航系統

GPS + IMU 全局導航，搭配沉積物沉降等待模式（SETTLING）。

🗺️ 作業模式

GOTO：單點 GPS 前往
SURVEY：割草機路線（行距 0.8m）
SETTLING：到達後等待 180秒（泥沙沉降）
RTH：返回出發點
EMERGENCY：超速 RTH（潮位超閾值）

⚠️ 待實作：Visual Servo

目前 YOLO 偵測到洞穴後，底座不會微調靠近，只記錄 GPS 座標。需要新增視覺伺服節點：
使用 center_world_m 引導底座前進，直到 distance < 0.35m 再切 PROBE。

✅ 沉積物沉降等待

新增 SETTLING 模式（SCAN_SETTLE_S = 180秒）：
螺旋停止後等待 3 分鐘讓泥沙恢復，再推進到下一個航點進行聲學掃描。
背景：螺旋擾動後聲學背景散射增加 10–20dB 長達 2–8 分鐘。

地面真值採集方法

💉

樹脂澆注（Resin Casting）

聚酯樹脂注入洞穴，固化後挖出，獲得完整 3D 洞穴幾何形狀。確認 1隻蝦：2個開口的比例。
用途：洞穴形態研究、設計聲學模型

🔧

吸水泵（Yabby Pump）

6 吋管徑吸水泵，30–50cm 深，每洞 3 次抽吸。確認率 70–85%（部分洞穴空置）。
用途：密度計數驗收，最常用方法

⛏️

手動挖掘

30×30cm 框，挖至 40cm 深。最準確但永久破壞基底，不可重複測量。
用途：生活史研究，特殊樣點

Gemini AI研究發現

透過 Gemini CLI 對專案所有程式碼與文件進行分析，識別出的關鍵缺口。完整報告：docs/gemini_research_briefing.md

🐛

Dead Zone BUG（已修正 ✅）

原：在距目標 1.0m 停下切換 PROBE，但臂展只有 0.57m → 永遠搆不到洞口。
修正：NAV_ARRIVE_DIST = 0.35m（survey_env.py、rl_agent_node.py）

🔢

Y型洞穴計數錯誤（已修正 ✅）

每隻蝦有 2 個開口，原系統每個開口算一隻 → 100% 高估族群數量。
修正：mission_logger.py 加入 30cm 配對邏輯，GeoJSON 分別記錄 total_openings / total_individuals。

📏

聲學穿透不足（文件化）

彰化粉砂 200kHz 穿透深度 20–30cm（Wang-gong），河口高濁度僅 5–8cm。蝦洞深度 30–60cm → 聲學只能確認洞口存在。深度量測需機械臂探針。

📳

振動擾動（已知限制）

馬達高頻振動在 3–5m 外即驅使蝦猴縮回洞內，導致機器人到達時洞穴已「空置」。無已發表行為恢復時間數據。

📡

無 Visual Servo（待實作）

YOLO 偵測到洞穴後，底座不使用視覺回饋微調位置，GPS 精度 ±1.5m 無法精確定位。需新增視覺伺服閉迴路節點。

⏰

沉積物沉降等待（已實作 ✅）

螺旋停止後加入 180秒 SETTLING 狀態（auto_navigate.py），等待泥沙擾動平息後再進行聲學掃描。

## 主題一：螺旋組件野外快速拆換機構

**查詢：** 螺旋組件目前是固定式，如何在 10 分鐘內完成野外換裝？考慮潮間帶泥沙、手套操作、20mm 不鏽鋼軸。

### Gemini 回應

針對潮間帶惡劣環境（泥沙、海水、腐蝕）與戴手套操作需求，Gemini 建議放棄細牙螺紋或緊配合，改用**幾何互鎖（Geometric Interlock）**搭配**大尺寸鎖定件**，推薦三個具體方案：

#### 方案一：貫穿銷與環狀保險銷（Clevis Pin & Linch Pin）—— 最強環境耐受度

農業機械與卡車最常用方案，結構最簡單，對泥沙完全免疫。

- **結構**：20mm 不鏽鋼軸端加工 8mm 貫穿孔；PVC 管內加裝 3D 列印（碳纖尼龍）或鋁製中心襯套，同樣對準開孔。
- **操作**：將螺旋組件套入軸心，插上貫穿銷，再扣上環狀保險銷。
- **優點**：戴厚手套也能操作；泥沙進入孔洞不影響功能，用水沖即可。
- **零件清單：**
  - 不鏽鋼 316 貫穿銷（Clevis Pin）：8mm × 40mm 或 50mm
  - 環狀保險銷（Linch Pin）：帶彈性環，防止銷釘滑脫

#### 方案二：D 型軸心 + 彈簧分度銷（D-Shaft & Spring Indexing Plunger）—— 免工具快拆

利用幾何形狀傳遞扭矩，彈簧銷僅負責定位，實現「一拉一抽」更換。

- **結構**：20mm 軸末端加工成 D 型軸（Flat face）或六角形；螺旋組件的襯套內孔做成對應形狀以傳遞 NEMA23 的扭矩；在襯套側面安裝重型彈簧銷。
- **操作**：拉起分度銷拉環，插入螺旋組件，旋轉至對位後彈簧銷自動卡入軸上的定位坑洞。
- **優點**：無需任何手持工具，拆裝僅需 5 秒。
- **零件清單：**
  - 不鏽鋼彈簧分度銷（Indexing Plunger）：型號 GN 607-ST-8-A（帶拉環）
  - 客製化 D 型襯套：6061 鋁材加工或高強度複合材料列印

#### 方案三：偏心把手夾緊環（Quick-Release Shaft Collar）—— 摩擦與定位混合

類似腳踏車座墊快拆把手。

- **結構**：螺旋組件端部為一開槽套筒，外圍套入帶有偏心連桿把手的夾緊環。
- **操作**：將螺旋組件推入 20mm 軸，按下把手鎖死；扳開把手即可抽出。
- **零件清單：**
  - 快拆式不鏽鋼夾緊環（Quick Release Shaft Collar）：內徑 20mm，型號 Ruland QRCL-20-F 或 MISUMI SCSQ20
  - 偏心把手（Clamping Lever）：GN 927 系列

#### 重要警告

> 所有金屬零件必須指定 **SUS316** 等級，不可使用 SUS304 或鍍鋅件，否則兩週內會因應力腐蝕開裂或生鏽卡死。

#### Gemini 建議總結

- **首選方案一（貫穿銷）**：現場泥沙極多且需要極高可靠性，最穩健。
- **次選方案二（D 型軸 + 彈簧銷）**：追求極速更換，軸端需加工。

## 主題二：5 台艦隊多機協調算法

**查詢：** 5 台機器人在 151 個網格（102.5 公頃）同時作業，如何避免碰撞、超音波干擾、重複調查？僅有 LoRa 433MHz 通訊。

### Gemini 回應

推薦採用**分散式拍賣演算法 + 地理圍欄預約制 + GPS 脈衝同步時分多址（TDMA）**的組合方案。

#### 1. 任務分配：分散式拍賣（Sequential Single-Item Auction）

- 靜態預劃分：151 個網格預先編號存入各機 Flash
- 標金計算：各機器人依當前座標與電量計算前往每個網格的「代價」（歐氏距離 + 轉向開銷）
- 異步宣告：機器人透過 LoRa 廣播「我接手網格 #10, #11, #12」，其他機器人收到後從本地可用列表刪除
- **優勢**：僅在切換目標時才通訊，極大降低 LoRa 負載

#### 2. 碰撞避免：動態預約緩衝區（Grid Occupancy）

- 虛擬保險桿：機器人不僅宣告當前所在網格，還需預約「下一個網格」
- 路徑排斥：若機器人 B 的規劃路徑進入機器人 A 已預約的網格，B 必須等待或繞道
- 心跳包：每 5~10 秒發送 `(ID, Lat, Lon, Target_Grid_ID)`；30 秒未收到視為斷訊，釋放預約區域

#### 3. 超音波干擾：GPS 1PPS 同步 TDMA

**核心方案**：利用 U-blox GPS 模組的 **1PPS（Pulse Per Second）**訊號作為全系統硬體同步觸發，**不需要透過 LoRa 同步時鐘**。

- 將 1 秒切分為 5 個 200ms 時隙（Slot）
- Robot 1：0ms～200ms 發射
- Robot 2：200ms～400ms 發射
- 以此類推...
- **200ms 緩衝**足夠讓 200kHz 聲波在水中傳播約 300m 並完全衰減

**TDMA 觸發邏輯（偽代碼）：**

```python
def timer_callback(self):
    now = self.get_clock().now().to_msg()
    ms_in_second = (now.nanosec // 1000000) % 1000
    my_slot = (self.robot_id - 1) * 200
    if my_slot <= ms_in_second < my_slot + 150:  # 留 50ms 緩衝
        self.trigger_ultrasonic_ping()
```

**LoRa 封包格式（8 bytes 壓縮）：**

```python
# [RobotID: 3bit][Status: 3bit][CurrentGrid: 8bit][TargetGrid: 8bit][Checksum: 8bit]
packet = struct.pack('BBB',
    (self.id << 5) | self.status,
    self.current_grid_index,
    self.target_grid_index
)
```

## 主題三：奧螻蛄蝦洞穴偵測補充方案

**查詢：** 現有 200kHz 超音波穿透深度 20~30cm，但洞穴深 30~60cm，評估 ERT、熱紅外、50kHz 超音波、機器學習融合的可行性與成本。

### Gemini 回應

#### 方案評估表

| 方法 | 穿透深度 | 解析度 | 野外自動化難度 | 預算（NT$） | 推薦指數 |
|------|---------|--------|--------------|------------|---------|
| **50kHz 低頻超音波** | 60cm+ | 低（需 T-SAFT 補償） | 低（替換現有硬體） | $10,000 | ★★★★★ |
| **熱紅外成像** | 0cm（僅表面） | 高（洞口定位） | 極低（被動式） | $30,000 | ★★★★☆ |
| **多感測融合 ML** | 60cm+（綜合） | 綜合優化 | 中（演算法門檻） | $15,000 | ★★★★★ |
| **ERT 電阻法** | 200cm+ | 中 | 極高（接觸式電極） | $50,000+ | ★★☆☆☆ |

#### 詳細評估

**50kHz 低頻超音波（最佳補充方案）：**
- 降低頻率以減少聲波在多孔隙介質中的散射與吸收損耗
- 50kHz 波長較長，可繞過微小沙粒，有效穿透深度可達 50~80cm
- 工業級 50kHz 換能器 + 驅動板：NT$5,000~12,000
- 缺點：解析度下降，需配合 T-SAFT 演算法補償；換能器體積較大

**熱紅外成像（快速洞口定位）：**
- 退潮後洞口溢出的地下水溫與地表泥灘溫差約 2~5°C
- FLIR 級別模組：NT$15,000~45,000
- 缺點：無法得知深度與內部結構，僅能確認「活洞口」位置

**ERT 地電阻法（不建議用於本系統）：**
- 潮間帶土壤含鹽水度極高，電阻率極低（低背景雜訊比）
- 需要插電極，對自動化機器人連續移動掃描難度極高
- 商業設備 NT$150,000~300,000；DIY 約 NT$30,000

**多感測融合策略（技術核心）：**
- 200kHz 鎖定 0~20cm 精細入口分支；50kHz 確認 30~60cm 主隧道（Hub）
- 利用現有 `burrow_cnn.py` 擴展為多模態輸入
- 需要 Jetson Orin Nano 計算單元：NT$15,000

#### 專家建議路徑

1. **硬體層**：保留現有 200kHz 做細節描繪，增設一只 50kHz 換能器做深層探測
2. **演算法層**：在 `acoustic_processor.py` 中導入多尺度 T-SAFT，將 50kHz 強反射信號作為 200kHz 深度特徵的 Weight mask
3. **環境適應**：利用紅外線在退潮初期快速定位高機率區域（Hotspots），引導機器人前往該點進行精準音波垂直掃描

## 主題四：公民科學整合方案

**查詢：** 如何整合漁民傳統知識（TEK）、iNaturalist、手機 APP、彰化縣政府/漁會合作，提升調查效率？

### Gemini 回應

#### 整合架構：熱點引導（Hotspot Guidance）

將公民科學數據轉化為機器人的「熱點引導」，而非僅作為靜態記錄。

**A. 漁民傳統知識（TEK）數位化：**
- 與彰化區漁會合作，舉辦「歷史產地繪製工作坊」
- 請資深採集者在數位地圖上標記 10~20 年前盛產區與目前退化區
- 將區域轉換為 `data/priority_cells.json` 中的權重參數；TEK 標記區的 `search_priority` 設最高

**B. iNaturalist + 專屬 APP 雙軌制：**
- 建立「彰化螻蛄蝦保育專案」於 iNaturalist，自動抓取彰化潮間帶的 *Austinogebia edulis* 記錄
- 開發輕量化標記 PWA（整合至現有的 `control/web_dashboard.py`），讓志工點擊畫面即可紀錄洞穴密集區座標並上傳帶比例尺照片

#### 資料品質控制（DQC）機制

| 來源 | 潛在問題 | QC 機制 |
|------|---------|---------|
| 漁民 TEK | 空間模糊、記憶偏差 | 比對 `tide_predictor.py` 歷史潮汐資料，排除海拔不符的錯誤標記 |
| iNaturalist | 物種誤認（如大眼蟹洞） | 調用 `burrow_cnn.py` 對上傳照片進行自動偵測，確認火山口狀特徵 |
| 志工 APP | 座標漂移、重複回報 | DBSCAN 算法將 5m 內重複回報聚合為單一熱點，計算信心分數 |

#### 代碼整合範例

修改 `data/mission_analyzer.py` 新增公民數據接收：

```python
def integrate_citizen_data(self, citizen_reports):
    for report in citizen_reports:
        if report['ai_confidence'] > 0.8:  # 經過 CNN 驗證
            self.update_priority_grid(report['lat'], report['lon'], increment=0.5)
```

#### 政府/漁會合作機制

- **彰化縣政府**：爭取納入「環境教育時數」認證，鼓勵大學生志工參與
- **彰化區漁會**：建立「回報獎勵制度」，漁民回報精準棲地資訊（經機器人驗證屬實後）可換取資材補貼或表揚
- **機器人驗證**：公民回報高密集區時，系統自動修改任務路徑，派遣機器人前往進行高解析度聲學採樣

#### 預期效益

- **搜索路徑縮短**：機器人直接前往志工標記的疑似區域，而非盲目掃描
- **時間尺度擴充**：漁民 TEK 提供歷史基線（Baseline），有助於分析棲地退化趨勢
- **社會認同**：讓當地居民從「資源利用者」轉變為「數據貢獻者」與「棲地守護者」

## 主題五：潮間帶長期維護策略

**查詢：** 材料選擇（316 不鏽鋼 vs 鋁合金 vs 工程塑料）、馬達保護、葉片磨損預測、維護排程、5 台艦隊備品清單。

### Gemini 回應

#### 1. 腐蝕防護與材料選擇：316L + HDPE/UHMW-PE 組合

- **傳動軸、緊固件、承重結構**：316L 不鏽鋼，進行鈍化處理（Passivation）增強抗點蝕能力
- **避免鋁合金**：即便硬質陽極處理，在螺栓連接處易產生電偶腐蝕（Galvanic Corrosion）；若必用，加裝犧牲陽極（Zinc Anodes）
- **螺旋葉片**：**UHMW-PE（超高分子量聚乙烯）優於 PVC**
  - PVC 在陽光 UV 下易脆化
  - UHMW-PE 具備極低摩擦係數與極高抗衝擊性，且不吸水

#### 2. NEMA23 步進馬達保護：正壓防護（Positive Pressure）

> IP68 密封在旋轉軸心處長期運作必然失效（海水因熱脹冷縮產生的負壓會被吸入）。

推薦策略：**氮氣/乾燥空氣充填 + 正壓維持**

- 將馬達封裝於定製鋁製/不鏽鋼殼體內
- 透過氣管持續輸入 **0.1~0.2 bar 微正壓**
- 軸封：使用**雙唇口骨架油封（Double Lip Rotary Seal）**，中間填充海洋級防水黃油（Marine Grease）
- 優點：即便密封微漏，氣體外洩會阻止海水侵入

#### 3. PVC/UHMW-PE 螺旋葉片磨損預測

磨損模型：L_wear = K × P × V × T（K: 磨損係數, P: 接地壓力, V: 旋轉速度, T: 作業時間）

**強制更換基準：**
- 葉片外徑減少 > 5%，或
- 邊緣厚度減少 > 30%

**預估更換週期：**
- 細泥質：500~800 小時
- 粗砂質：150~300 小時

**最佳化建議**：在葉片邊緣加裝可更換式「犧牲邊緣條（Sacrificial Edge）」

#### 4. 預防性維護排程

**每次作業後（Post-op）—— 關鍵：去鹽分**

1. 淡水沖洗：重點沖洗旋轉軸密封處、散熱片與感測器窗口（防止鹽分結晶磨損）
2. 正壓檢查：確認馬達殼內氣壓是否正常維持
3. 目視檢查：確認是否有漁網、海藻纏繞螺旋軸

**每週維護（Weekly）**

1. 潤滑補給：透過油嘴（Zerk Fitting）注入新黃油，將舊的含沙黃油擠出
2. 螺栓緊固：檢查 316 不鏽鋼螺栓是否有鬆脫（螺旋振動大）

#### 5. 5 台艦隊備品清單（基於 20% 維修週轉率）

| 項目 | 數量 | 備註 |
|------|------|------|
| NEMA23 馬達總成 | 2 套 | 含預裝正壓殼體與密封 |
| 螺旋葉片單元（UHMW-PE） | 5 組 | 易磨損件，模組化設計便於快速更換 |
| 雙唇口軸封（Oil Seals） | 20 個 | 消耗最快，建議每 100 小時更換 |
| 316L 緊固件包 | 2 套 | M5/M6 螺栓、墊圈（極易在泥水中遺失） |
| 驅動皮帶/聯軸器 | 4 件 | 緩衝動力衝擊的犧牲件 |
| 海洋級防水黃油 | 5 桶 | 每日消耗品 |

**額外建議**：在 `control/teleop_node.py` 加入**馬達電流監測回饋**，當螺旋被硬質異物卡住導致電流異常飆升時，自動觸發「反轉脫困」程序。

## 總結：前五大可行性建議（影響力/成本比排名）

### #1 — 螺旋葉片快拆：貫穿銷方案（影響力 ★★★★★ / 成本 ★）

**行動：** 在 20mm 不鏽鋼軸端加工 8mm 貫穿孔，搭配 SUS316 Clevis Pin + Linch Pin。

- 估計成本：NT$200~500 / 組（標準農機零件）
- 工時：軸端加工 + 配合孔車削，外包約 NT$1,000~2,000
- 好處：野外換裝縮短至 3 分鐘內（相較現有固定式可能耗時 30 分鐘以上）
- 設計更動：端蓋內側需增加中心定位襯套（3D 列印碳纖尼龍即可）

### #2 — 葉片材料升級：UHMW-PE 取代 PVC（影響力 ★★★★★ / 成本 ★★）

**行動：** 下一批生產的螺旋葉片改用 UHMW-PE 板材車削或 CNC 加工。

- 估計成本：UHMW-PE 板材 NT$1,000~3,000 / 組（比 PVC 貴約 1.5~2 倍）
- 好處：使用壽命從 150~800 小時延長至估計 500~2,000 小時；UV 耐受更佳
- 行動項目：修改 `docs/procurement_bom_v2.md` 中的葉片材料規格

### #3 — 50kHz 雙頻超音波補強（影響力 ★★★★★ / 成本 ★★★）

**行動：** 增設一只 50kHz 換能器，搭配多尺度 T-SAFT 算法實現雙頻融合。

- 估計成本：NT$5,000~12,000（換能器 + 驅動板）
- 好處：有效穿透深度從 20~30cm 提升至 50~80cm，接近洞穴完整深度（30~60cm）
